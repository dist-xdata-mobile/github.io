# 用户手册—数慧模型管理系统 V1.0【DME】

作者:Dist 时间： 2020-10-16

# 1 文档介绍

# 1.1 文档目的

为了使用户更快的熟悉数慧模型管理系统的操作方法，规范操作步骤，尽量缩短用户的上手时间，特此编写此用户手册。本手册结合实际操作，配以详细的文字说明和图片展示，确保用户高效快捷的使用本软件。

# 1.2 读者对象

系统使用人员

# 2 系统运行环境

# 2.1 硬件环境

本系统运行的硬件环境要求如下：

名称 详细信息
CPU 英特尔 Core i5 以上
内存 8GB 以上
硬盘 200GB 以上空间
网卡 100Mbps

# 2.2 软件环境

本系统运行的软件环境要求如下：

名称 详细信息
操作系统 Windows 7（64bit）及以上
安装软件 谷歌浏览器

# 3 基本术语

术语 说明
模型 模型是为了实现某一个业务逻辑功能将一系列的数据源、算法、组件等串联在一起的工作流。
数据源 数据源指通过注册方式注册到模型评价软件的数据源列表中的数据，可在模型构建中通过被拖拽连线的方式进一步配置使用。
算法 接收特定的输入参数，实现某些特定的功能逻辑，并输出预设的结果。
组件 组件是对数据和方法的简单封装。
任务 这些任务可以分给不同的程序进行处理，也可以一个程序处理很多个任务，任务的处理逻辑是通过模型定义的。

# 4 数慧模型管理系统

数慧模型管理系统，简称 DME，采用分布式计算架构：分布式缓存 redis、分布式的存储 mongodb、分布式的任务调度，以 Task 并行计算的方式充分利用多核的资源，以达到对外提供高效、分布式的数据计算能力。建设了一个集模型管理、模型开发、模型计算于一体的管理平台，实现了模型的跨平台应用和可扩展性。

数慧模型管理系统针对国土空间规划监测、预警和评估的模型需求，通过注册数据源和相关算法，拖拽数据源、算法、组件等构建适用于不同场景的各类应用模型，设置不同的参数，执行模型计算，生成相应运行结果并存储到数据库中，实现模型的统一管理和应用，从而为国土空间规划编制的基础分析评价、规划审查、专项评估等提供模型计算支撑。

# 4.1 算法管理

对已实现并封装好的 DLL 文件进行注册管理，包含算法注册、算法删除、算法元数据编辑等。

# 4.1.1 算法注册

可将编译好的算法文件，注册到系统的算法目录中。已注册的算法可在模型构建中通过被拖拽连线的方式进一步配置使用。

# 4.1.2 算法管理

以列表的方式，展示已注册算法的基本信息。对无需使用的算法，可进行删除。对算法名称、算法分组、算法版本、算法说明等算法元数据可进行灵活编辑。

l 支持根据关键字搜索并筛选出符合搜索关键字的算法。

l 支持算法分组的添加、修改、删除。

l 支持查看算法说明和参数说明。

l 支持编辑算法的名称、分组、说明。

l 支持从算法分组中删除该算法。

l 支持同时删除多个算法。

l 支持同时对多个算法移动分组。

l 支持同时下载多个算法。

l 支持以压缩包的形式下载该算法。

# 4.2 数据源管理

针对算法所需的各类数据源进行统一管理，包括数据源注册、数据源删除、数据源元数据编辑等功能。

# 4.2.1 数据源注册

可将模型中需要用到的数据，注册到系统的数据源列表中，支持 MDB、GDB、SDE、Oracle 等多种数据库类型。已注册的数据源可在模型构建中通过被拖拽连线的方式进一步配置使用。

# 4.2.2 数据源管理

以列表的方式，展示已注册数据源的基本信息。无需使用的算法，可进行删除。对数据源名称、数据源分组、数据源简介等数据源元数据可进行灵活编辑。

l 支持数据源分组的添加、修改、删除。

l 支持查看数据源说明和参数说明。

l 支持编辑数据源的名称、分组、说明、内容等。

l 支持从数据源分组中删除该数据源。

l 支持同时删除多个数据源。

l 支持同时对多个数据源移动分组。

l 支持同时下载多个数据源。

l 支持根据关键字搜索并筛选出符合搜索关键字的数据源。

l 支持以压缩包的形式下载该数据源。

# 4.3 组件查看

对封装在系统中的实用组件进行查看。

l 支持根据关键字搜索并筛选出符合搜索关键字的组件。

l 支持对打开组件的基本配置信息进行修改并保存。

# 4.4 模型构建

支持通过拖拽连线的方式设计模型流程，通过对数据源、算法（含运行参数）、结果存储形式进行配置，构建出模型。构建好的模型支持进行注册，已注册的模型将会进入模型管理界面予以统一管理。设计过程中支持查看算法信息，并支持对已建模型进行编辑修改。

# 4.4.1 模型目录管理

以目录树的形式，显示已注册的数据源、算法和组件，供构建模型时进行拖拽。

l 支持对模型分组的添加、删除、修改。

l 支持批量删除该模型下的所有模型子模型。

l 支持在模型目录下以及模型下新建子模型。

l 支持模型的复制、粘贴。

l 支持查看接口名称、简介、地址，可点击地址进一步查看详情。

l 支持编辑模型分组、名称、图片、简介、标签。

l 支持根据关键字搜索模型、子模型。

# 4.4.2 模型构建

通过拖拽连线数据源和算法实现模型设计，通过调整参数和存储等进行配置，以画布形式，展示模型的构建。

l 支持新建目录、模型、子模型。

l 支持展示所有算法，并支持拖拽配置。

l 支持展示所有数据源，并支持拖拽配置。

l 支持展示所有组件，并支持拖拽配置。

l 支持展示所有模型，并支持拖拽配置。

l 支持编辑算法节点的节点名称、节点说明，以及配置算法的参数。

l 支持复制节点和粘贴节点并保存节点配置。

l 支持编辑数据源节点的节点名称、节点说明。

l 支持编辑组件节点的节点名称、节点说明。

l 支持编辑模型节点的节点名称、节点说明。

l 支持保存当前模型子模型建模状态和参数配置。

l 支持注册已构建好的模型子模型到“模型管理”模块，作为模型产品进行管理。

l 支持将配置好的模型或子模型整体打包导出（适用于换库，模型不需重新配置）。

l 支持将打包的模型或子模型整体导入（适用于换库，模型不需重新配置）

# 4.5 模型管理

针对模型进行查询、运行和注销管理。支持查询模型的输入输出参数以及接口详情，支持通过运行输入参数启动模型进行计算，支持模型注销。

模型管理中支持针对具体的模型进行模型运行参数配置以及任务的新建，支持定时任务和即时任务的创建。模型运行的结果将在运行监控管理中进行统一管理。

此外，还支持查看模型详细信息，包括参数信息和接口信息，支持调整到模型构建页面，对模型设计进行修改。

# 4.5.1 查看模型

可查看模型的分组、简介、接口列表等。跳转到“模型构建”模块，显示浏览对应模型子模型的建模流程。

# 1.1.1 运行模型

将已构建并注册好的模型，执行运行，并跳转至监控页面，查看实时计算状态。

# 1.1.2 注销模型

将不需要的模型进行注销。

# 1.1.3 模型产品管理

对建好的模型产品进行查看、排序并修改产品展示信息等。

l 支持按照模型目录筛选模型子模型。

l 支持根据关键字搜索并筛选出符合搜索关键字的模型子模型。

l 支持下载作为当前模型封面的图片。

l 支持替换作为当前模型封面的图片。

l 支持按模型修改更新的时间排序。

l 支持按模型的名称排序。

# 4.6 运行监控

支持对模型运行的情况进行监控管理，包括任务状态、任务执行周期、运行时间和运行结果情况，同时提供任务日志查看功能，可查看某次任务运行状态及结果。

# 1.1.4 任务管理

以列表的形式展示模型运行情况，可查看历史运行情况和当前运行情况。通过按钮可进入各任务详情内页，查看任务状态、执行周期、运行时间和运行结果情况，支持对单条任务记录的增加删除等。

l 支持启动运行任务。

l 支持结束正在运行的任务。

l 支持禁用定时任务，使定时任务失效。

l 支持启用定时任务，使定时任务恢复定时执行。

l 支持删除单条或多条任务。

l 支持编辑任务名称、任务描述和任务类型（定时任务/即使任务）。

l 支持根据关键字搜索并筛选出符合搜索关键字的任务。

l 支持下载存在后台的本地结果数据。

l 支持同时运行选中的多个任务。

l 支持同时结束选中的多个任务。

# 1.1.5 任务日志

提供任务日志查看功能，可查看某次任务运行的详细状态及结果。

l 支持查看任务的名称、子模型、上次运行时间、结束时间、运行状态等信息。

l 支持产看任务的运行日志，查看任务执行失败的原因，任务成功的过程记录。

# 1.1.6 运行统计

提供针对模型的运行统计应用，包括任务状态统计和模型执行统计。

l 支持统计自定义时间段内任务运行成功、失败、中止等的个数。

l 支持将自定义时间段内具体模型执行的次数由多到少统计排列。
